# Item 22. 인터페이스는 타입을 정의하는 용도로만 사용하라

## 개요

인터페이스는 클래스가 어떤 행위를 할 수 있는지 알려준다. 그리고 인터페이스는 오직 이 용도로만 사용해야한다.

## 안티 패턴 - 상수 인터페이스

- 정의
    - static final 필드로 가득 찬 인터페이스

- 상수는 내부 구현에 해당하므로 사용되는 클래스 내부에 위치하는 것이 좋다
    - 상수를 외부에 공개할 것이라면?
        - 연관된 클래스나 인터페이스 내부에 위치하도록 하는 방식
            - ex) Integer.MIN_VALUE
        - Enum 사용
        - 인스턴스화 할 수 없는 유틸리티 클래스를 사용하는 방식
